QUICK INSTALL (5 minutes)
═══════════════════════════════════════════════════════════════

STEP 1: Check Prerequisites
───────────────────────────

✓ Chrome Version 138+ 
  → Check at: chrome://version
  → If older, update Chrome

✓ Native Operating System (NOT WSL2)
  → Windows 10+, macOS 13+, or dual-boot Linux
  → WSL2 users: test on Windows host instead

✓ Hardware Requirements
  → 8GB+ RAM
  → 4GB+ VRAM (GPU memory)
  → 10GB+ free disk space

✓ Download AI Models
  → Open: chrome://on-device-internals
  → Check status, download if needed
  → Wait for "Ready" status


STEP 2: Test Environment
──────────────────────────

Open Chrome DevTools Console (F12) and paste:

  console.log('Summarizer:', 'Summarizer' in self);
  console.log('Translator:', 'Translator' in self);
  console.log('LanguageModel:', 'LanguageModel' in self);

All should show "true". If false, see troubleshooting below.


STEP 3: Prepare Icons
──────────────────────

Create 3 icon files in the icons/ folder:
  • icon16.png (16x16 pixels)
  • icon48.png (48x48 pixels)
  • icon128.png (128x128 pixels)

Quick options:
  → Screenshot emoji: https://emojipedia.org/page-facing-up/
  → Placeholder: https://via.placeholder.com/16/2563EB/FFF?text=DG
  → Design your own with any image editor


STEP 4: Load Extension
───────────────────────

1. Open Chrome
2. Go to: chrome://extensions/
3. Enable "Developer mode" (top-right toggle)
4. Click "Load unpacked"
5. Select the DocuGuide folder
6. Done! Extension should appear in toolbar


STEP 5: Test It
────────────────

1. Click DocuGuide icon → "Open Sidebar"
2. Look for status: should say "✓ Ready"
3. Go to any webpage with text (try Wikipedia)
4. Select a paragraph
5. Click "Summarize" button
6. Should see a summary appear!

🎉 If it works, you're done! Start using DocuGuide!


TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

Problem: Status shows "✗ Error"
────────────────────────────────

Solution:
  1. Open Settings → "Test API Availability"
  2. Go to chrome://on-device-internals
  3. Verify models show "Ready"
  4. Restart Chrome completely
  5. Reload extension


Problem: "APIs Not Available"
──────────────────────────────

Solutions:
  → Update Chrome to 138+
  → Download models at chrome://on-device-internals
  → If on WSL2: test on Windows host instead
  → Check hardware: need 8GB RAM, 4GB VRAM


Problem: On WSL2 - Nothing Works
─────────────────────────────────

Reality: WSL2 is virtualized, Chrome AI needs native OS

Your options:
  → Open Chrome on Windows (not in WSL2 terminal)
  → Access Windows files: /mnt/c/Users/YourName/
  → Load extension from Windows path
  → Or test on Mac/Linux dual-boot


Problem: Icons Not Showing
───────────────────────────

Solution:
  → Create all 3 icon files (16, 48, 128 pixels)
  → Must be PNG format
  → Must be in icons/ folder
  → Must be named exactly: icon16.png, icon48.png, icon128.png
  → Reload extension
